--- title: "Why does auto deploy not work when I use a new tag on a Docker Hub image?" id: 1145 draftstate: inactive deleted: false source: https://success.docker.com/@api/deki/pages/1145/contents tags: - tag: "article:topic" - tag: "component:hub" - tag: "expires:nextrelease" - tag: "product:cloud" - tag: "stage:reviewed" --- {% raw %}

Currently, the auto-deploy feature only triggers when you push images with the latest tag. If you push other tags, it will not trigger an automated re-deploy for the service.

You will need to manually stop/start the service again.
